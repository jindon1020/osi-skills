# 飞书云文档全内容提取指南

本 skill 旨在提供一套标准化的流程，确保 Agent 能够深入提取飞书云文档中的所有视觉和文本信息，特别是传统提取方法容易忽略的“画板”内容。

## 核心提取流程

### 1. 初始分析与环境确认
- **目标识别**: 确认目标 URL 是否为飞书文档（`feishu.cn` 或 `larksuite.com`）。
- **访问控制**: 使用 `browser_subagent` 打开页面。如果遇到登录墙，需提示用户提供 Cookie 或访问权限。
- **结构扫描**: 初始阶段，通过 MCP 获取文档结构快照。识别文档中的块（Blocks）类型，特别是定位：
    -   `text`: 标准文本块（通过 MCP 直接读取）。
    -   `whiteboard` / `canvas`: 画板块（需标记位置，后续调用浏览器抓取）。
    -   **注意**: 忽略 `video` (视频) 块及外部/内部跳转链接，无需进行深度探索。

### 2. 文本与主体结构提取 (MCP 优先)
- **策略**: 优先调用 **飞书云文档 MCP** 相关工具（如获取文档 Markdown 内容或 Blocks 列表）来获取文档的主体框架。
- **优点**: 速度快、文本还原度高、层级结构清晰。
- **文本抓取**: 获取具有层级结构的文本（H1, H2, 列表等），保留原始缩进和引用。
- **占位符记录**: 在 MCP 返回的内容中，记录 `whiteboard` 或 `canvas` 块的出现位置，作为后续浏览器抓取的锚点。
- **多媒体处理**: 记录图片和表格的元数据。

### 3. 画板（Canvas/Whiteboard）深度提取
画板内容无法通过简单的 DOM 抓取，需遵循以下步骤：
1.  **定位画板**: 在页面中找到画板容器。
2.  **视口对齐**: 自动滚动页面使画板完全显示在视口中心。
3.  **精细化缩放 (Zoom-in)**: 对于密集复杂画板（如用户提供的示例），应指导 `browser_subagent` 放大局部视图（推荐 150%-200%），以确保文字清晰度。
4.  **分块截图采样 (Tiled Capture)**: 
    -   如果画板内容丰富，采用分块模式截图。
    -   优先截取核心逻辑区、文字密集区。
5.  **视觉描述（Vision Analysis）**: 
    -   通过视觉模型分析截图内容。
    -   识别画板中的：**节点、连接线、思维导图结构、流程图、手绘草图**。
    -   将这些视觉信息转换为结构化的文字描述。

### 4. 复杂图文的合并与输出
- **序列重组**: 按照文档的 Blocks 顺序，将文本、图片描述、画板描述重新排版。
- **Markdown 生成**: 
    -   画板内容应以 `> [画板内容描述]: ...` 的形式插入。
    -   图片使用标准的 Markdown 语法引用或保存路径。
- **质量检查**: 验证是否遗漏了折叠块（Callout）或隐藏内容。

## 使用建议

-   **分段抓取**: 对于超长文档（超过 50 个 blocks），建议分段执行提取，防止浏览器上下文过大。
-   **画板精度**: 如果画板内部文字极小，应指导 `browser_subagent` 进行“放大查看”后再截图。
-   **视频忽略**: 飞书文档中的视频内容不需要抓取或理解，直接忽略，不生成相关描述。
-   **链接处理**: 忽略文档中引用的外部或内部链接内容，**不要**深入点击或探索链接指向的页面，仅保留当前的文本或画板内容。
