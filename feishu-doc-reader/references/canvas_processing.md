# 画板（Canvas/Whiteboard）提取技术细节

提取飞书画板内容时，由于其基于 Canvas 或复杂的 SVG 渲染，直接读取 DOM 文本通常不完整。请遵循以下技术策略：

## 1. 画板类型识别
飞书文档中可能存在两类“画板”：
-   **嵌入式画板**: 直接在文档流中显示。
-   **全屏画板（多维表格视图）**: 需要点击跳转或展开。

## 2. 浏览器自动化指令
在使用 `browser_subagent` 时，应执行以下操作以获取最高质量的画板数据：

### 自动滚动与对齐
```javascript
// 示例：将画板元素滚动到中心并等待渲染完成
const canvasBlock = document.querySelector('.docx-whiteboard-container');
if (canvasBlock) {
  canvasBlock.scrollIntoView({ behavior: 'smooth', block: 'center' });
}
```

### 获取结构化线索
虽然无法直接抓取图形，但可以尝试抓取画板中的浮层文字（如果有）：
-   检查 `.canvas-text-layer` 或类似的组件。
-   抓取画板工具栏中的“导出”按钮状态。

## 3. 高清缩放与分块采样 (针对超复杂画板)
当遇到如“产品全景图”、“系统架构图”等高密度内容时，应执行：
1.  **缩放对齐**: 针对画板的核心模块，使用浏览器脚本或模拟按键（如 `Cmd +`）放大内容至 **150%-200%**。
2.  **分块视口 (Viewport Tiling)**: 
    -   将画板划分为 2x2 或 3x3 的逻辑块。
    -   依次平移画板，确保每个局部内容的文字清晰可见后再截图。
3.  **多图联立识别**: 告诉视觉模型各张图片之间的空间顺序（如“第一行左一”、“中心块”），以便模型构建完整的逻辑链路。

## 3. 视觉模型 (Vision) 提示词策略
在分析截取的画板图片时，使用以下 Prompt 模版：
> “请详细分析这张飞书画板截图。识别并列出其中的所有文字。如果是一个思维导图，请将其转换为 Markdown 级别的层级结构。如果是一个流程图，请描述各个步骤之间的逻辑关系（如：A -> B [判断条件] -> C）。请尽可能保留颜色、图标所代表的含义。”

## 4. 常见问题处理
-   **画板过大**: 如果画板内容超出一个屏幕，指导 Agent 使用平移（Pan）工具并进行多次截图拼合。
-   **懒加载**: 确保滚动到画板位置后，等待至少 2 秒让矢量图形完全加载和锐化。
-   **权限受限**: 如果画板显示为“无权查看”，记录该位置并告知用户，不要跳过，以保持文档结构的完整性。
